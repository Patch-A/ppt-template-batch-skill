# Agent Runtime Rules

## Aily 兼容边界

该目录本身就是 Aily/飞书智能体 skill 根目录：根目录必须有 SKILL.md，其他内容只使用 references/ 下的 Markdown 和 JSON。不要要求 skillhub 识别 manifest.json、engine/、requirements.txt 或 Codex 专用目录。

飞书/Aily 直接使用原生幻灯片能力完成 PPT 创建、复制页面、放置图片和导出。该运行路径不安装 Python、python-pptx、Playwright、OpenAI SDK，也不需要任何模型 API Key。仓库中的 Python/PPTX 脚本只服务于桌面版，不是 Aily skill 的启动依赖。

## 任务识别

从用户请求提取 template、mode、country、procurement_need、buyer_count、records 和 allow_page_expand。如果没有 mode，根据模板和用户描述判断：一页一个企业用 buyer_board，一页 6 个企业用 buyer_briefing，其他情况用 generic。

## 模板拆解摘要

开始填充前必须记录：

- 幻灯片总数、封面页和重复内容页。
- 每个可填充元素的页码、文本框或表格位置、字段名。
- 固定元素：Logo、页脚、免责声明、装饰线、底图、封面标题结构。
- 字体、字号、颜色、粗细、行距、对齐方式、文本框尺寸和图片框比例。
- 页面不足时复制哪一页，复制后哪些字段需要清空或替换。

用户标记为固定、红框或“不需要修改”的元素永远不改。

## 样式保留边界

- 模板自带元素只改文字，不重设样式。标题、底部提示语、页脚、免责声明、蓝色头栏、装饰组、Logo 和底图均属于固定元素。
- 禁止新建文本框替换原标题或页脚，也禁止为了“统一样式”重新设置 `font.name`、`font.size`、`bold`、`color`、阴影、描边、行距、位置或文本框属性。
- 替换时在原 shape、原 paragraph、原 first run 内写入；如果旧文字拆成多个 run，清空其余 run，但保留第一个 run 的原始格式。只有用户明确批准的内容字段才允许 style override。

## 企业资料核验

使用飞书智能体自带搜索能力时，至少交叉检查企业官网、企业介绍页、新闻/项目页或政府/交易/行业来源中的两类证据。记录：

- 企业全称和官网域名。
- 企业是实际使用方、项目采购方、分销商、系统集成商还是仅制造商。
- 采购需求与企业业务场景的具体关联。
- 来源链接、访问日期和无法确认的字段。

“企业生产某产品”不等于“企业采购该产品”。没有采购匹配依据时降低优先级或剔除。

完成核验后先保存标准 buyers 数据，再进入 PPT 排版。飞书/Aily 模式不要调用需要 OPENAI_API_KEY 或其他模型 Key 的自动检索脚本；确定性引擎只接收已经由智能体搜索完成的 buyers.json、records.json 和图片路径。

## 图片规则

1. Logo：优先官网页眉、品牌资源页、结构化 Logo 或官方媒体资料；必须匹配当前企业本身，母公司页面里的子品牌、业务单元、认证徽章和合作品牌 Logo 均不可替代企业 Logo；裁成透明或紧凑比例，和正文文本框左边缘对齐。
2. 右侧图：优先官方产品图、工厂/应用场景中含产品的图片，再用官网产品页面截图。
3. 图片框固定后使用裁剪填充，不拉伸变形；按模板框的宽高比和安全边距计算，禁止越界或遮挡文字。
4. 官网打不开时按顺序尝试公开地图、官方社媒、搜索结果图片和行业来源；所有非官网素材标记来源。
5. 没有真实素材时才用原生生图，生成产品或应用场景配图，不生成企业 Logo，不把 AI 图声称为官网实拍。

## 文案规则

- 买家看板简介按模板要求补足中文字符，保持企业名称、业务范围、采购场景和关键证据。
- 买家商情简介必须包含企业名称，只介绍企业情况，贴近模板示例长度，不写成采购清单。
- 采购品类按模板固定格式输出，并写到可采购的具体产品层级，例如“切菜机、切肉机”，不能只写“商用厨房设备、食品加工设备、中央厨房系统”等大类；具体产品必须能由企业产线、门店、项目、维护或分销场景支持。
- 内容太长优先压缩重复表达、复制内容页或调整文本框允许的段落，不要随意缩小字号。
- 买家看板的采购产品行和简介行应按实际换行数动态调整高度，短内容不得保留大块空白；企业、国家、官网等固定短行保持模板高度，页眉和页脚不移动。

## 失败处理

- 搜索失败：保留任务并输出待核验企业及失败原因，不编造官网和采购事实。
- Logo 失败：使用空占位或官方文字标识，不截取随机网页区域冒充 Logo。
- 产品图失败：尝试图片搜索、地图、官方社媒，最后使用原生生图并标记 generated。
- PPT 导出失败：输出结构化错误、失败阶段和可重试输入；不要只回复“生成失败”。

## 模型选择建议

如果运行环境支持模型选择，优先使用当前可用的高能力推理/多模态模型；GPT-5.5 / ChatGPT(OpenAI) 仅作为推荐示例，不是硬性依赖。飞书原生模型、用户指定模型或当前可用模型均可继续使用，不要求额外 API Key。

## 已验证的兼容性修复

以下规则来自飞书 Agent 实际生成任务中的回归问题，属于每次填充和导出的必检项：

1. **填充前清空旧文字**：模板表格或文本框可能把一段文字拆成多个 run。不能只替换第一个 run；应先清空所有段落中的所有 run，再写入一个干净的文本 run，并恢复模板原有字体、字号、颜色、粗细和段落格式。导出后扫描旧模板关键词，发现残留即判定失败。
2. **动态调整内容行**：企业、国家、官网和页眉页脚保持模板高度；采购产品和简介按实际字符数、换行数、列宽、字号和内边距计算高度。短文本不得保留大块空白，长文本不得被截断，也不得挤压固定页脚。
3. **复制完整内容页**：模板页数不足时，必须复制完整的内容页或使用平台的“复制页面”能力，连同表格、图片、装饰元素和页面关系一起复制；不能用只复制空白母版布局的新增页面替代。
4. **Logo 与右图分开核验**：Logo 必须匹配当前企业主体，不能因文件名含 `logo` 就采用认证徽章、政府标识、子品牌或业务单元标识；右侧图片单独按产品图、应用图、官网图的优先级筛选，不能从右图反向截取 Logo。
5. **无可靠 Logo 不造假**：官网、公开地图、官方社媒和图片搜索均无法确认时，保留空占位或使用企业文字标识，并在来源报告中写明失败原因；不得把随机网页区域或 AI 生图冒充企业 Logo。
6. **固定样式专项校验**：逐页比对内容页标题和底部提示语与模板原始 run 的字体、字号、颜色、粗细、阴影/描边和位置；发现变化时回到原 shape/run 修复，不通过新建文本框补救。

详细现象、根因、示例和版本记录见 `buyer-board-workflow-changelog.md`。
