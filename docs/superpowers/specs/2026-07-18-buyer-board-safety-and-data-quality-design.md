# Cross-Preset Safety And Quality Design

日期：2026-07-18

## 目标

修复最近代码审查发现的买家看板数据真实性、资产抓取安全和缓存正确性问题，并把可复用的质量保障扩展到通用 PPT、Buyer Briefing、一图全解、Feishu/Aily 和控制台。增加可重复的回归测试，保持现有 CLI、控制台接口、Feishu/Aily skill 包和 PPT 模板版式兼容。

## 范围

### 资产抓取安全

- 只允许 `http` 和 `https` 资源。
- 对用户或模型提供的 URL 做基础规范化，并阻止本机、回环、链路本地和私有网络地址。
- 对跟随重定向后的最终地址重新执行同样的安全校验；默认不接受跨主机重定向到非官方资源。
- Python、curl、Data URI 三条下载路径都执行响应大小上限，超限时在内存和磁盘写入前终止。
- 保留现有每买家和全局时间预算。

### 资产缓存与报告

- Logo 和站点图分别判断缓存是否有效；只命中其中一项时，另一项仍然重试。
- 缓存保存成功结果和可重试的失败状态，失败不阻塞下一次运行。
- 正常、跳过和失败记录使用同一组报告字段，便于控制台和后续工具消费。

### 买家资料真实性

- 简介长度补齐只使用中性企业描述，不添加未经证实的采购能力、采购计划或合作意愿。
- 产品字段始终根据买家证据收敛，即使全局采购需求只有 1-3 项也不能无条件复制给所有买家。
- 保留现有“需核实具体设备”降级语义；证据不足时不得生成确定性采购结论。

### Logo 识别

- 品牌匹配只使用候选文件名、alt 和 class 等候选自身字段，不使用页面 URL 作为拒绝依据。
- 将 `rgb`、`color`、`mark`、`symbol`、年份等描述性后缀视为中性词。
- 只有没有品牌交集，或命中明确的竞争品牌/子品牌规则时才拒绝候选。

### 通用 PPT 引擎

- 在 `layout-config` 中优先使用 shape 名称、占位符角色和字段 ID，shape index 只作为兼容回退。
- 统一检查字段缺失、模板残留文字、文本溢出、图片越界、页数异常和 PPTX 可打开性。
- 批量任务按记录隔离失败，输出原子化，并生成逐记录和总任务报告。
- TXT、Markdown、CSV、XLSX、JSON、DOCX 导入继续保留原始来源文本和用户指令，字段清洗结果必须可追踪。

### Buyer Briefing 预设

- 每个品类页严格维护 6 个买家槽位，自动分页并清理不足槽位中的旧模板文字。
- 统一简介长度、采购品类前缀、空字段和字段缺失报告。
- 输出实际买家数、页数、溢出、残留文字和跳过记录。

### 一图全解预设

- `content-map` 和表格数据增加版本、必填字段和缺失 shape 报告。
- 文本容量按中文、英文、换行、字号和文本框容量估算，不只使用总字符数。
- 表格行高使用明确的最小高度和内容容量策略，不依赖把行高归零来触发未知的 Office 行为。
- 增加 dry-run，先报告超长、缺 shape、表格越界和可能溢出，再决定是否输出。

### Feishu/Aily 技能包

- 统一 generic、buyer_board、buyer_briefing 三种模式的输入输出协议、来源字段和人工核验状态。
- 明确原生搜索、图片搜索、幻灯片编辑能力不可用时的降级路径，不伪造来源或图片。
- 构建 ZIP 时检查版本、必需文件和禁止携带的桌面运行时依赖，保持包与桌面引擎职责分离。

### 控制台与基础设施

- 子进程执行增加有界超时和可识别失败阶段。
- 默认只允许回环地址，避免无认证 API 暴露到局域网。
- 项目、记录、布局和报告采用原子写入，API 响应保持统一结构，模型密钥始终脱敏。
- 增加跨预设测试夹具和最小 PPTX 输出检查，避免某个预设的规则污染其他模式。

## 不在本轮范围

- 控制台任务队列的完整持久化、断点续跑和远程认证。
- PPT 渲染服务替换或完整的视觉回归基础设施；本轮只增加可执行的最小输出检查和容量预检。
- API provider 路由重构和模型升级。

## 数据流

1. 根据模式选择通用字段映射或对应预设规则，不把买家规则应用到 generic 或一图全解。
2. 买家研究输出经过产品、简介和来源字段规范化。
3. 资产抓取先规范化 URL，再执行安全校验、下载上限和候选过滤。
4. 资产结果按 Logo/站点图独立写入缓存和买家记录。
5. 所有模式通过统一 preflight 生成字段、容量、残留、页数和输出完整性报告。
6. 回归测试覆盖纯函数、缓存短路、资源下载边界、三种预设和最小 PPTX 产物。

## 错误处理

- 安全校验失败返回可识别的拒绝原因，不回退到不受限的下载路径。
- 超过响应大小限制时释放临时文件并记录 `response_too_large`。
- 单个买家资产失败不影响其他买家；全局预算耗尽的记录仍写入完整报告 schema。
- 证据不足继续保留买家，但标记人工核实，不填充未经证实的采购事实。

## 测试策略

- 使用标准库 `unittest`，避免新增测试依赖。
- 先写失败测试，再逐项实现：Logo 中性后缀、页面路径误拒绝、缓存缺失资产重试、短采购需求收敛、简介中性补齐、资源大小限制、通用映射校验、Buyer Briefing 六槽位、一图全解 dry-run 和统一报告字段。
- 使用最小 PPTX 夹具分别验证 generic、buyer_board、buyer_briefing 和一图全解，不依赖 PowerPoint COM 才能完成基础回归。
- 运行全量 Python 语法编译、测试套件、CLI `--help` 冒烟、Feishu ZIP 合约检查和最小 PPTX 生成检查。

## 兼容性与迁移

- 保持现有 JSON 字段名和 CLI 参数；仅新增报告字段或内部状态。
- 旧缓存缺少新字段时按缓存未命中或空状态处理，不删除用户已有资产。
- 默认资产模式和浏览器回退开关保持不变。
